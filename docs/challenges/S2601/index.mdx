## 项目描述

LiteRT (之前称 TensorFlow Lite) 作为 Google Edge AI 的核心推理框架，广泛应用于移动端和嵌入式设备。目前 LiteRT 对 RISC-V 架构的支持尚不完善，未能充分发挥 **RVV (RISC-V Vector Extension) 1.0** 的向量并行能力，本赛题旨在填补这一空白，推动 RISC-V 在边缘 AI 领域的生态建设。

## 产出以及评分要求

1. 功能适配：为 LiteRT 添加 RISC-V 64 位支持，确保其正确编译通过并运行。

2. 算子优化：利用 RVV 1.0 指令集，优化 LiteRT CPU 内核中核心算子，包括 GEMM、卷积、激活参数及常用逐元素运算。

3. 文档撰写：记录针对 RISC-V 架构的适配/优化思路，测试方案，测试报告等，用于组委会验证参赛选手的优化成果。

4. 验证模型：基于自带的 benchmark_model 工具运行 mobilenet_v1_1.0_224 和 mobilenet_v1_1.0_224_quant [模型获取链接](https://github.com/tensorflow/models/blob/master/research/slim/nets/mobilenet_v1.md)。

**注意事项**

请确保遵循相关的开源协议和版权规范。

参赛者需要自行准备开发环境，组委会后续会对参赛者提供 A210 远程调试环境。

## 赛题验证

- 验证平台: &copy; 知合计算 A210 系列开发板

### 性能验证

| 验证项 | 要求 |
|--------|------|
| **推理延迟（Latency）** | 要求低于 < 110 ms ，包含：avg（平均）、p50（中位数）、p95（尾延迟）|
| **吞吐率（Throughput）** | throughput = FPS = batch_size x 1000 / latency_ms |
| **稳定性** | 须包含：std （标准差）、p95 / avg 比值 |
| **内存占用** | Overall footprint 值 |
| **启动时间** | Model initialization 值 |
